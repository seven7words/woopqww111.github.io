AssetBundle Workflow

AssetBundle 工作流程

balabla的介绍听从下面的步骤即可

Assigning Assets to AssetBundles

分配资源（Assets）给AssetBundles

分配指定的资源给AssetBundles，有如下几个步骤：
1.从你的项目里选择你想分配给bundle的asset。
2.在Inspector里测试你的object
3.在Inspector的底部你会看见一个（我实在不知道怎么描述这个）其实就是一个方块。用来分配AssetBundle和后缀的
4.左边的下拉框是用来分配AssetBundle的名字的，右边的下拉框是用来分配后缀的名字的
5.点击左边的下拉框看是不是None覆盖了已经注册的AssetBundle的名字
6.如果没有什么被创建。你可以看那个下拉框的上部的清单
7.点击New可以创建一个新的AssetBundle
8.就是命名是有规范的，你看你自己想怎么创建都行，并且／这个符号是用来分开文件夹和assetbundle的。只有最后一个是assetbundle。其他在／这个前面的都是文件夹
9.一旦你选择或者创建亿个AssetBundle名字，你可以重复上面的步骤去分配或者创建后缀名。如果你想的话，后缀名可以不参与打包。

Build the AssetBundle

创建AssetBundles

这个步骤呢是只有在编辑器 下可以，所以呢，你得创建一个叫做Editor的文件夹，并且在里面编写一个Editor脚本类似于：

using UnityEditor;
public class CreateAssetBundles
{
[MenuItem("Assets/Build AssetBundles")]
static void BuildAllAssetBundles()
            {
            string assetBundleDirectory = "Assets/AssetBundles";
            if(!Directory.Exists(assetBundleDirectory)
        {
        Directory.CreateDirectory(assetBundleDirectory);
            }
    BuildPipeline.BuildAssetBundles(assetBundleDirectory,         BuildAssetBundleOptions.None, BuildTarget.Standalone);
    }
    }

看代码就可知其意。没啥好解释的。

Upload AssetBundle to off-site storage

每个用户的使用方法是不一样的。所以Unity可以告诉你怎么做，如果你计划上传你的AssetBundle给第三方主机，按照下方的实例可以模仿一下，如果你严格的在本地开发，并且企图将所有的AssetBundle放在本地磁盘里。那么跳过下面的步骤。

Loading AssetBundles and Assets

为了用户可以在本地内存中加载，使用AssetBundles.LoadFromFile API。就像下面代码一样。
(官网给的是js，我个人用C#所以我翻译了一下）

public class LoadFromFileExample : MonoBehaviour
{    
    void Start() {
            AssetBundle myLoadedAssetBundle = AssetBundle.LoadFromFile(Path.Combine(Application.streamingAssetsPath, "myassetBundle"));
        if (myLoadedAssetBundle == null)
        {
            Debug.Log("Failed to load AssetBundle!");
            return;
        }
        var prefab = myLoadedAssetBundle.LoadAsset<GameObject> ("MyObject");

        Instantiate(prefab);
    }
}

LoadFromFIle就是从路径中的bundle文件读取。
如果你自己托管你的AssetBundles彬哥切你需要去下载他们到你的游戏里，你应该使用UnityWebRequest ，下面是例子：

IEnumerator InstantiateObject()
{
    string uri = "file:///" + Application.dataPath + "/AssetBundles/" + assetBundleName;        
    UnityEngine.Networking.UnityWebRequest request = UnityEngine.Networking.UnityWebRequest.GetAssetBundle(uri, 0);
    yield return request.Send();
    AssetBundle bundle = DownloadHandlerAssetBundle.GetContent(request);
    GameObject cube = bundle.LoadAsset<GameObject>("Cube");
    GameObject sprite = bundle.LoadAsset<GameObject>("Sprite");
    Instantiate(cube);
    Instantiate(sprite);
}

GetAssetBundle(string,int)就是根据uri得到assetbundle的位置，和你想下载的版本号。你可以使用本地的uri。也可以使用主机的。

UnityWebRequest 是一个特殊的用来处理这类需求的API。DownloadHandlerAssetBundle。用来从当前的request得到AssetBundle。
不管你使用哪类方法，最终都需要一个入口从AssetBundle中获取范型的对象。LoadAsset。然后。没什么了。
